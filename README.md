# ttcAm
Tair-Trafic-Capture_And_Modification

Данный репозиторий предназначен для обхода блокировок по DPI(L7,SNI), но вы также можете настроить его под свои руки(Например создать свой собственный Firewall или родительский контроль).
Проект будет написан на языке программирования Python. 

Библиотеки:
```
pip install pydivert
pip install lupa
```

К сожалению, ttcAm не умеет обходить блокировки по IP, и вам придется покупать прокси или 3 буквы сервер.
# Сейчас
В нынешнее время часто используется DPI для глубокого анализа ваших пакетов, а если система заметит, что вы заходите на запрещенный сайт(roblox.com,youtube.com,discord.com), она отреагирует RST-пакетами и выдаст вам Timed Out. Каждый пакет содержит свою TLS версию, SNI и так далее, а РКН хочет знать все о своих пользователях, но им мешает такой домен, как encryptedsni.com. Он шифрует ваш SNI, обманывая систему DPI и реальный пакет успешно проходит на запрещенный ресурс. Многие браузеры добавляют такую технологию, а РКН пытается узнать структуру таких зашифрованных SNI пакетов.

# Методы десинхронизации
Программа на данный момент имеет только 3 метода десинхронизации(fake,split,disorder), но в будущем список будет обновляться. 
## Fake
  Fake - это метод десинхронизации, который посылает фейк-пакеты с имитацией настоящего пакета или рандомных байтов. Данный метод очень хорош в том, что он может атаковать DPI своими фейк-пакетами дабы ваш реальный запрос пролетел мимо DPI. 
  
  Главный минус Fake - это нужно подобрать правильный TTL(Time-To-Live). Иначе вы можете оглушить DPI(Молодец, ты пробил DPI), но и сервер, он может воспринять это, как дос атаку и отклонить ваш запрос(RST-Пакет)
## UDP_len
  UDP_len - это метод десинхронизации, который отвечает за увеличие udp пакета. Если пакет больше 1200 байтов(явный признак udp пакета) то к нему можно применить данный метод. В конце payload пакета добавляется лишний мусор, который увеличивает максимум на 300 байт
## Split
  Split - это метод десинхронизации, который позволяет разбить ваш пакет на определенные части. Для чего он используется? Когда запросы могут быть слишком тяжелы для сервера, иногда пакет может быть "фрагментирован" дабы серверу было легко переваривать ваш запрос.

  Не рекомендуется использовать Split метод в нынешнее время. DPI системы научились склеивать фрагментированые пакеты, но на процесс они тратят очень много оперативной памяти. 

## Disorder
  Disorder - это улучшенная версия метода десинхронизации split. Её основная модификация состоит в том, что она отправляет части пакета не по порядку, а в разнобой. Он может разделить пакет на 2 части(получим 1,2,3), а затем сделать из них кашу для DPI, но для сервера это будет легкотня ведь он умеет склеивать части пакета поставив их в очередь(Out-Of-Order)

  Не рекомендуется резать пакет на много частей. Вы так запутаете DPI и сервер. Хватит разделения на 1 байте
# Fooling modes
Программа также имеет режимы для дополнительного обмана. Доступные варианты: badsum,badseq,md5sig,datanoack
Стратегия будет загружать мод обманывания из файла fooling.mode. Можно будет также отключить fooling mode

Подробнее в файле guide\fooling.txt

# Фильтры по L3 и L7
L3 фильтр позволяет модифицовать только пакеты IPv4 или IPv6. Вы такэе можете отключить IPv6 на своем компьютере нажав win+r/ncpa.cpl/*сеть к которой вы подключились*/Свойства/*уберите галочку с IP версии 6 (TCP/IPv6)*/ОК
Вы также можете проверить через сайт https://api64.ipify.org
L7_filter позволяет сортировать пакеты по протоколам(tls,http,udp,unknown). Раньше использовалась функция l7_filter из питон файла, но это снижало производительность. 
На данный момент используется встроенный .dll файл l7_filter написанный на C. C код обеспечивает скорость выполнения задач

# Поддержка .lua стратегий
С версии 0.2.1 моя программа поддерживает .lua код. Конкретно используется LuaJIT(Just-In-Time Compiler)
За ядро у нас отвечает ttcAm.py, а раньше все методы десинхронизации писались по отдельному и это занимало много времени
Все аргументы прописаны в ядре(packet_data.args*аргумент*)
Отдельная благодарность автору zapret2 bol-van за идею.

# Поддержка C кода
С версии 0.2.7 программа поддежрживает .C коды. Вы можете сделать свой собственный .dll файл для модификации ядра(уже сделан l7_filter.dll для улучшения оптимизации)
