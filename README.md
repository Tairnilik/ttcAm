# ttcAm (Tair-Traffic-Capture_And_Modification)

**ttcAm** - это модульный инструмент для анализа и модификации сетевых пакетов на уровне ядра Windows. Проект разработан для обхода глубокого анализа пакетов (DPI) методами десинхронизации на уровнях L3, L4 и L7. Благодаря гибкой архитектуре, систему можно адаптировать под задачи локального файрвола, родительского контроля или тестирования сетевой безопасности.

> **Ограничение:** Программа работает исключительно с сигнатурным и структурным анализом трафика (L7/SNI). Блокировки по IP-адресу данный софт обойти не может — для этих целей вам потребуется использовать сторонние прокси-серверы или зарубежные VDS.

## Архитектура проекта

Утилита построена на базе четырех технологических слоев, что позволяет совместить гибкость разработки и высокую скорость обработки трафика:
1. **Ядро (`ttcAm.py`)** — координирует перехват трафика, управляет конфигурациями и связывает компоненты системы.
2. **Низкоуровневый перехват (`WinDivert`)** — драйвер режима ядра, выполняющий физическую остановку и внедрение пакетов в сетевой стек Windows. Для работы требуются права администратора.
3. **Логика десинхронизации (LuaJIT / `.lua`)** — сценарии обработки вынесены в изолированные скрипты. Это избавляет от необходимости перекомпилировать ядро при изменении стратегий. Взаимодействие обеспечивается библиотекой `lupa`.
4. **С-модули (`.dll`)** — критические для производительности задачи вынесены в нативный C-код. Например, фильтрация протоколов на уровне L7 реализована в `l7_filter.dll`, что минимизирует задержки (overhead) при обработке потока данных.

## Методы десинхронизации трафика

Система манипулирует структурой TCP/UDP-соединений, чтобы сбить с толку парсеры DPI, сохраняя при этом легитимность пакетов для целевого сервера.

### 1. Fake (Поддельные пакеты)
Метод отправки фейковых пакетов (например, имитирующих TLS Client Hello или содержащих случайный байтовый шум) с заниженным временем жизни (TTL). 
* **Механика:** Фейковый пакет доходит до цензора (DPI), заставляя его сбросить контекст отслеживания или посчитать сессию закрытой. До целевого сервера этот пакет не долетает из-за истечения TTL, благодаря чему реальный запрос проходит следом без препятствий.
* **Особенность:** Требует точной подстройки параметра TTL под конкретного провайдера, иначе сервер может зафиксировать атаку и разорвать соединение (RST).

### 2. Split (Фрагментация TCP)
Разбиение исходного TCP-пакета на несколько мелких сегментов.
* **Механика:** Исторически фрагментация использовалась для снижения нагрузки на узлы сети. Для DPI обработка множества мелких осколков требует значительных ресурсов оперативной памяти для их сборки и анализа.
* **Особенность:** Современные системы DPI часто умеют пересобирать фрагментированные пакеты в буфере. Метод эффективен в комбинации с другими стратегиями.

### 3. Disorder (Перемешивание сегментов)
Модифицированная и более агрессивная версия метода Split.
* **Механика:** Пакет делится на части, которые отправляются в сеть с нарушением естественного порядка (в разнобой). Защитные системы DPI путаются в потоке и пропускают данные. При этом целевой сервер легко собирает исходный запрос, так как стек TCP/IP на уровне ОС по умолчанию поддерживает обработку пакетов, пришедших не по порядку (Out-Of-Order).
* **Рекомендация:** Достаточно выполнить разделение на уровне первого байта. Избыточное дробление на множество частей может перегрузить очередь сервера.

### 4. UDP_len (Модификация UDP)
Специфический метод для работы с UDP-трафиком (например, QUIC).
* **Механика:** Если размер UDP-пакета превышает пороговое значение в 1200 байт (типичный признак хэндшейка), к концу его полезной нагрузки (payload) принудительно дописывается мусорный padding размером до 300 байт, что искажает сигнатуру для фильтров DPI.

## Дополнительные режимы обмана (Fooling Modes)

Для усложнения анализа трафика на стороне цензора реализованы низкоуровневые модификации заголовков. Параметры загружаются динамически из файла конфигурации `fooling.mode`:
* `badsum` — генерация некорректной контрольной суммы (Checksum) для фейковых пакетов. Сетевое оборудование провайдера обработает пакет, но конечный сервер его отбросит.
* `badseq` — подмена порядкового номера (Sequence Number) в TCP-заголовке.
* `md5sig` — включение опции TCP MD5 Signature, которая часто отбрасывается конечными серверами, но обрабатывается промежуточными фильтрами.
* `datanoack` — отправка данных без флага подтверждения.

## Фильтрация трафика

### Уровень L3 (Сетевой)
Программа поддерживает фильтрацию пакетов протоколов IPv4 и IPv6. 

*Примечание: Если на вашем узле некорректно работает стек IPv6, его рекомендуется отключить в настройках сетевого адаптера:*
1. Нажмите `Win + R`, введите `ncpa.cpl` и нажмите Enter.
2. Кликните правой кнопкой мыши по сети, к которой вы подключены, и выберите **Свойства**.
3. Уберите галочку с пункта **IP версии 6 (TCP/IPv6)** и нажмите **ОК**.
4. Проверить текущий стек можно через внешние сервисы (например, api64.ipify.org).

### Уровень L7 (Прикладной)
Встроенный модуль на Си осуществляет высокоскоростную сортировку пакетов по сигнатурам протоколов:
* `TLS` (включая анализ полей Client Hello и обработку зашифрованных SNI/ESNI).
* `HTTP` (парсинг заголовков Host).
* `UDP` (выделение QUIC/STUN потоков).
* `Unknown` (неизвестный тип трафика).
